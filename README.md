# Sparta-Shopping Project

## 프로젝트 개요

이 애플리케이션은 다음 주요 기능을 제공합니다:
1. **인기 검색어 및 검색 기능**: 검색 기능과 함께 인기 검색어를 추적하는 기능 구현.
2. **`LIKE` 쿼리를 사용하는 검색 API**: 특정 컬럼을 기준으로 `LIKE` 조건을 사용하여 검색하는 기능.
3. **검색 API에 In-memory Cache 적용**: `spring-boot-starter-cache`를 사용해 캐싱을 적용한 검색 API 구현.
4. **Redis 캐시 (선택 사항)**: 검색 API v2에서 Redis를 활용하여 캐시를 적용.

## 주요 기능

### 1. 인기 검색어

- 인기 검색어 목록을 조회할 수 있는 API를 제공합니다.
- 이 기능은 사용자들이 가장 많이 검색한 키워드를 트래킹하거나 트렌드 검색어를 파악하는 데 유용합니다.

### 2. `LIKE` 쿼리를 사용한 검색 API

- 사용자가 특정 컬럼을 기준으로 항목을 검색할 수 있는 기능을 제공합니다.
- `LIKE` 조건을 사용하여 텍스트 기반 검색을 구현합니다.
- 검색 결과는 페이지 단위로 제공되어, 한 번에 많은 데이터를 처리할 수 있습니다.

**API 상세 정보**:
- **v1 API**: `/api/v1/boards/search`
    - 캐시가 적용되지 않은 일반 검색 API입니다.
- **v2 API**: `/api/v2/boards/search`
    - **In-memory Cache**가 적용된 개선된 검색 API입니다.
    - 캐시를 활용하여 자주 요청되는 검색에 대한 성능을 향상시킵니다.

### 3. 캐시 적용

- Spring AOP 방식의 `@Cacheable`을 사용하여 In-memory 캐시를 구현합니다.
- **캐시를 적용한 이유**:
    - 캐시는 같은 검색 쿼리에 대해 반복적으로 데이터베이스에 접근하는 것을 방지합니다.
    - 데이터베이스 부하를 줄이고, 자주 조회되는 검색에 대해 빠른 응답을 제공합니다.

### 4. Redis (선택 사항)

- v2 API에서는 Redis를 활용하여 캐시를 적용할 수 있습니다.
- **Redis를 사용하는 이유**:
    - Redis는 빠른 속도의 인메모리 데이터 저장소로, 자주 요청되는 데이터를 캐시하여 성능을 크게 향상시킬 수 있습니다.
    - 대규모 애플리케이션에서도 높은 성능을 유지할 수 있도록 지원합니다.

## 팀 역할

- **김기홍**: 회원 인증 및 인가
- **신기환**: S3 통합 및 CI/CD
- **이규정**: 상품 관리
- **석연걸**: 주문 및 주문 항목 관리

## 기술 스택

- **백엔드**: Spring Boot
- **캐시**: In-memory 캐시 (Spring Cache) 및 Redis (선택 사항)
- **데이터베이스**: JPA, QueryDSL
- **CI/CD**: GitHub Actions
- **S3**: Amazon S3

## 검색 API에 캐시를 적용한 이유

캐시는 자주 요청되는 검색 데이터를 빠르게 제공할 수 있게 도와줍니다. 동일한 검색 쿼리에 대해 반복적으로 데이터베이스를 조회하는 대신 캐시된 데이터를 활용하여 성능을 향상시킬 수 있습니다. 이는 특히 트래픽이 많은 애플리케이션에서 매우 유용하며, 사용자에게 빠른 응답을 제공하여 경험을 개선할 수 있습니다.
